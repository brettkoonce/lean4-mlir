## Reusable infra + gotchas discovered this session
- **Offline codegen validation without a GPU:** `iree.runtime` (local-task CPU
  driver, in the lean4-jax venv) RUNS a CPU-compiled vmfb standalone — so attention/
  optimizer codegen is numerically validatable on CPU while GPUs are busy. The
  project FFI is GPU-only, but `iree.runtime` isn't. This is what made flash/rope
  validation possible mid-session. Pattern: `scripts/flash_probe_check.py`.
- **Validate-then-emit:** every new op got a numpy reference FIRST (shampoo jewel,
  flash fwd+bwd, rope) validated to ~1e-15, THEN the emitter matched it to fp32.
  The emitters were near-first-try because the math was pinned. Keep doing this.
- **`flash-probe` exe** (`demos/MainFlashProbe.lean`) emits standalone SDPA/rope
  modules for validation — modes: (default) flash-fwd, `bwd`, `dense`, `rope`.
- **Lean gotcha:** multi-line record-update `{ x with a := …,\n b := … }` fails to
  parse — keep the whole `with` on ONE line (hit it 3× this session).
- **Codegen gotcha:** a new `Layer` field with a default fills SHORTER patterns with
  the DEFAULT value (not a wildcard), so exhaustive matches break AND param-count
  walks silently use the default. When adding a field: update the constructor,
  ALL pattern sites (fwd body, TRAIN sig walks × how-many, EVAL sig walks — these
  are SEPARATE and easy to miss → `%W1` type-clash), backward, optimizer update,
  Spec totalParams/shapesBA/heInitParams. `posEmb` touched ~10 sites.
- **flash block size** `flashBlock(n)` = 128 if n%128==0 else 64/n; the fwd and bwd
  must agree (both recompute from n).
